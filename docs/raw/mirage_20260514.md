Mirage is **a Unified Virtual File System for AI Agents**: a single tree that mounts services and data sources like S3, Google Drive, Slack, Gmail, and Redis side-by-side as one filesystem.

AI agents reach every backend with the same handful of Unix-like tools, and pipelines compose across services as naturally as on a local disk. It's a simulated environment, agents see one filesystem underneath. Any LLM that already knows bash can use Mirage out of the box, with zero new vocabulary.

```ts
const ws = new Workspace({
  '/data':   new RAMResource(),
  '/s3':     new S3Resource({ bucket: 'logs' }),
  '/slack':  new SlackResource({}),
  '/github': new GitHubResource({}),
})

await ws.execute('grep alert /slack/general/*.json | wc -l')
await ws.execute('cat /github/mirage/README.md')
await ws.execute('cp /s3/report.csv /data/local.csv')
```

## About

- **One filesystem, every backend.** Every service speaks the same filesystem semantics, so agents reason about one abstraction instead of N SDKs and M MCPs.
- **Multiple resources, one filesystem:** RAM, Disk, Redis, S3/R2/OCI/Supabase/GCS, Gmail/GDrive/GDocs/GSheets/GSlides, GitHub/Linear/Notion/Trello, Slack/Discord/Telegram/Email, MongoDB, SSH, and more.
- **Familiar bash tools across every mount.** Agents reuse Unix-like tools instead of learning a new API per service.
- **Portable workspaces:** clone, snapshot, and version your environment.
- **Python and TypeScript SDKs** for embedding in FastAPI, Express, browser apps.
- **Works with:** OpenAI Agents SDK, Vercel AI SDK, LangChain, Pydantic AI, CAMEL, OpenHands.
- **Lightweight CLI + daemon** plugs into Claude Code and Codex.

## Architecture

AI Agent and Application → Mirage Bash and VFS → Dispatcher & Cache → Infrastructure and Remote

## Quickstart (Python)

```python
from mirage import Workspace
from mirage.resource.gdocs import GDocsConfig, GDocsResource
from mirage.resource.ram import RAMResource
from mirage.resource.s3 import S3Config, S3Resource
from mirage.resource.slack import SlackConfig, SlackResource

ws = Workspace({
    "/data":  RAMResource(),
    "/s3":    S3Resource(S3Config(bucket="my-bucket")),
    "/slack": SlackResource(SlackConfig()),
    "/docs":  GDocsResource(GDocsConfig()),
})

await ws.execute("cp /s3/report.csv /data/report.csv")
await ws.execute("grep alert /s3/data/log.jsonl | wc -l")
ws.snapshot("demo.tar")
```

## Agent Frameworks

Mirage integrates as a sandbox/tool layer:
- **OpenAI Agents SDK**: `MirageSandboxClient` plugs a Workspace as sandbox
- **Vercel AI SDK**: `mirageTools(ws)` exposes workspace as typed tool set
- **LangChain, Pydantic AI, CAMEL, OpenHands** adapters available

## Cache

Two-layer cache on every Workspace:
- **Index cache** — listings and metadata (TTL-based)
- **File cache** — object bytes
- Pluggable backends: RAM (default, 512 MB) or Redis (shared across workers)
