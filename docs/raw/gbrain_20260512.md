# GBrain

## Overview

**GBrain** gives AI agents a persistent brain. Built by the President and CEO of Y Combinator (Garry Tan) to run his actual AI agents. Production deployment: 17,888 pages, 4,383 people, 723 companies, 21 cron jobs running autonomously.

## Key Features

- **Hybrid Search** — Vector + keyword + RRF (Reciprocal Rank Fusion)
- **Self-Wiring Knowledge Graph** — Typed entity links (`attended`, `works_at`, `invested_in`, `founded`, `advises`) with zero LLM calls
- **PGLite** — In-browser Postgres, no server needed, database ready in 2 seconds
- **34 Skills** — Organized by a resolver that tells the agent which skill to read for any task
- **MCP Server** — 30+ MCP tools via stdio, HTTP with OAuth 2.1 support
- **Minions** — Durable Postgres-native job queue for deterministic background work
- **Durable Agents** — Subagent runs survive crashes with replay from last committed turn
- **Skillify** — Turn any fixed bug into a permanent skill with a 10-item audit
- **Storage Tiering** — Keep bulk content (tweets, articles, transcripts) out of git
- **BrainBench** — P@5 49.1%, R@5 97.9% on 240-page corpus

## Architecture

```
Brain Repo (git) ←→ GBrain (Postgres + pgvector) ←→ AI Agent (34 skills)
```

The repo is the system of record. GBrain is the retrieval layer. Human always wins — edit any markdown file and `gbrain sync` picks up changes.

## Install

On an agent platform (OpenClaw or Hermes Agent) or standalone:

```bash
git clone https://github.com/garrytan/gbrain.git && cd gbrain && bun install && bun link
gbrain init
gbrain import ~/notes/
gbrain query "what themes show up across my notes?"
```
